React App Development

The Problem

Online grocery shopping has exploded in popularity in recent years. As a result, clean and easy-to-use online grocery websites are becoming increasingly important. The application below aims to solve this need via a simple, crisp, and intuitive interface for online grocery shopping.

I set out with a goal to create a web application using the React framework that provides for a seamless shopping experience.

User Research/User Needs

Based off of my personal experiences, interactions with classmates, and interviews with frequent users of online shopping websites, the following were all listed as must haves for any website:

  • Users are able to add items and remove items from a dynamic shopping cart.
  • Filtering functionality allows users to find the items most important to them, and the sorting functionality can quickly sift through the cheapest items in each category.
  • The shopping cart provides users with the ability to see in real time the cost of the grocery items they are looking to purchase.

The Website

Design Decisions

The color scheme was chosen with clear visibility in mind. The dark black and blue on white background is very simple, but very readable for users.

The cards are laid out to follow the user's natural scan patterns. Items are sorted from left to right, then from top to bottom. Buttons to add and remove are at the bottom of the card, matching the ending point of user scan patterns.

System status is always kept visible. Displays always show the user which sort and filtering criteria are being used to keep users informed as to the status of the application.

All cards and displays are kept consistent across the application and the aggregated shopping cart section. Users never have to learn multiple formats, and so can extract the information they need from each item with minimal effort.

The system speaks the user's language. Phrases such as "Add to Cart" use common language used in physical grocery stores - avoiding system oriented language.

Users can undo actions and errors quickly and efficiently. The "Add to Cart" and "Remove from Cart" features are available in both the shopping cart section and the main items section, reducing the amount of scrolling that the user has to do to accomplish their desired tasks.

Users are provided instructions for any items that are not clearly visible. There is a marker at the very top indicating that the shopping cart appears at the bottom of the page, minimizing any potential confusion.

The design is minimalist and avoids overloading the user with information. Every single piece of information displayed on the website is directly relevant to the users' goals.

Conclusion/Parting Thoughts

Creating a user-friendly application using stateful programming is a skill that all designers should have. This project allowed me to connect the usability principles I have learned into an actual product.

It pays dividends to fully flesh out the architecture of any application before coding. Even a simple application such as the one above has a lot of moving parts, and it makes life infinitely easier to have a clear cut plan before diving into the code.

It's worth it to have redundancy in some features of an application. For example, users should have the ability to add and remove items from cart from both the landing page and the shopping cart page. This minimizes scrolling and makes the design as intuitive as possible.
