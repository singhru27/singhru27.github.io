Responsive Redesign

The Problem

Well designed websites are the cornerstone behind any business, organization, or nonprofit. A properly designed website can make the difference between a site with high traffic-flow/high monetization potential and one that is stagnant.

What makes a good website "tick"? Today, we'll be finding out as we explore the webpage for a local investment banking business - a website that would be well served with many improvements.

The Original Website

Usability/Accessibility Issues and Research

Usability Issues

Orange font makes it hard to see headers, with text often blending into the white background

  • Color palette is generally not coordinated and clashes with one another

Broken links on the navigation bar. User can’t navigate outside of this specific webpage and has to physically type in the homepage URL

  • No indication that the navigation bar even allows for navigation to other pages (there is no change on hover)
  • No indication of where we are on the webpage
  • Disappears once we scroll down, defeating the purpose of the navigation bar in the first place

Unclear what the purpose of Reuter’s business on the right-hand side of the page is

  • Compression into such a small space requires significant scrolling

Font sizing is inconsistent (header size is not larger than subheader size), making the hierarchy hard to visualize

  • Font size of the actual text is extremely small, and the webpage does not dynamically change font size on smaller screens
  • Font coloring is inconsistent, leaving users wondering whether “is this shade of blue a link or just a random shading?”

Unclear how the page should be read: Our first intuition is to read left to right, however the page is designed such that the first column is read top-to-bottom, then the second column is read. Grouping in general is very poor

  • Action buttons are not at the end of scan patterns

Left alignment is off (not all text is properly aligned). Capitalization is inconsistent

Images are blurry and hard to see, irrelevant and ugly blue box right-justified along the image makes the entire page very jarring

On smaller pages, the entire page just becomes smaller. The font size decreases and everything compresses

  • Becomes essentially unreadable on tablets and phones due to the lack of responsiveness

Accessibility Issues

Very low contrast makes the page difficult to read for those with visual impairments

  • Specifically within the text on the top left blue box within the page, the text is a pale white whereas the background is blue

Layout tables are used throughout the document. Since the content is not tabular in nature, screen readers might interpret them as data tables

Alternative text for pictures is completely unrelated to what the picture actually is

  • A picture of a map has the alternative text label of “BlackBridge Partners LLC”

The page has no heading structure, making it hard to follow the structure of the page when using a screen reader

  • Screen reader usually calls out heading levels, so users can determine which level of the content they are on. When using the “VoiceOver” screen reader, there is no such information and as such it is difficult to figure out what is going on

The page has no language listed, which is needed to aid screen readers

Conclusion/Parting Thoughts

Accessibility is something that is so easy to consider, but so often overlooked. Small features like alt tags make a huge difference for those who rely on screen readers and other accessibility based software packages.

An unbelievably large number of companies and organizations feature old, outdated, and inaccessible websites. I was surprised to find how many of the local businesses in my area had websites that did not follow many (if any) visual design principles.

It can be deceptively easy to introduce dependencies into a website that cause usability issues. One broken link or misplaced image source file can ruin the aesthetic of an entire website.

I had a ton of fun making this project and learning the ins and outs of the Bootstrap frameworks. I'm looking forward to embarking on some more adventurous website redesigns in the future!
